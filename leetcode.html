// 06 — Pattern Cheatsheet

When to Use What

The most valuable skill is pattern recognition — knowing which technique to apply within 30 seconds of reading a problem.

Two Pointers
Use when → sorted array, find pair/triplet with target sum
One pointer at start, one at end. Move based on condition. O(n) instead of O(n²).
Two Sum II3SumContainer With Water
Sliding Window
Use when → subarray/substring of fixed or variable size
Expand right pointer, shrink left when condition breaks. Avoids recomputing overlapping parts.
Max SubarrayLongest SubstringMin Window
Binary Search
Use when → sorted input, or search on answer space
Halve the search space each step. Works on arrays AND on answer ranges (e.g., min/max problems).
Search RotatedKoko EatingFind Min
BFS / DFS
Use when → graph traversal, shortest path, components
BFS = level by level (shortest path). DFS = go deep (cycle detection, topo sort).
Number of IslandsCourse ScheduleClone Graph
Dynamic Programming
Use when → optimal substructure + overlapping subproblems
Break into subproblems, cache results. Top-down (memo) or bottom-up (tabulation).
Climbing StairsCoin ChangeLCS
Backtracking
Use when → all combinations/permutations/subsets needed
Try a choice, recurse, undo. Exhaustive search with pruning to cut dead ends early.
SubsetsPermutationsN-Queens
Heap / Priority Queue
Use when → kth largest/smallest, top-K, streaming data
Min-heap for kth largest. Max-heap for kth smallest. O(n log k) beats sorting.
Kth LargestTop K FrequentMerge K Lists
Fast & Slow Pointers
Use when → linked list cycle, middle of list
Slow moves 1 step, fast moves 2. If they meet → cycle. Where they meet → cycle start.
Linked List CycleMiddle of ListHappy Number
Monotonic Stack
Use when → next greater/smaller element, span problems
Maintain stack that's always increasing or decreasing. Pop when current breaks order.
Daily TemperaturesLargest RectangleNext Greater

// 12 — FAQ

Beginner FAQs

Common questions every beginner asks — answered honestly.

How long will it take to get good at LeetCode? ▼
For most consistent beginners, 3–4 months of daily practice to solve Mediums comfortably. Knight badge typically takes 5–8 months. There's no shortcut — consistent daily practice is the only path.
C++ or Python — which should I use? ▼
C++ is faster (important in contests with tight time limits) and forces you to understand memory better. Since you're already using C++, stick with it — it's the right choice for competitive programming and gives you an edge in execution speed.
I'm stuck on a problem for over 45 minutes — what do I do? ▼
After 30–45 minutes of no progress, read the editorial approach (not the code). Understand the idea, close the solution, then implement it yourself from memory. Don't copy-paste — re-derive it. This is where learning actually happens.
What ratio of Easy / Medium / Hard should I solve? ▼
Ideal ratio: 20% Easy, 65% Medium, 15% Hard. Don't spend too much time on Easy (they don't appear in real interviews much). Mediums are the bread and butter. Hards are for contest rating and top-tier company prep.
Does LeetCode rating actually matter for placements? ▼
Not directly — companies don't ask for it. But the Knight badge is a strong signal of ability on your profile. More importantly, grinding for rating forces you to learn the hard patterns that appear in actual interviews.
How many problems before applying for internships? ▼
Minimum: 150–200 problems with strong Medium coverage. Comfortable: 300+ with all core patterns covered. Quality over quantity — being able to solve any Medium from scratch matters more than your count.
Is random grinding or pattern-based learning better? ▼
Pattern-based learning is far more effective. Group problems by topic: "all sliding window", then "all BFS", etc. This builds recognition far faster than random or difficulty-based ordering. Use the study plans — they're already organized this way.
