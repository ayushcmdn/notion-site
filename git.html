Setup & Configuration
Pehli baar Git install karne ke baad ye karo — ek baar hi karna hota hai
git config --global user.name
Beginner
Apna naam set karo Git ke liye. Ye naam har commit mein dikhega.
📌 Kab use karein: Pehli baar Git install karne ke baad
$git config --global user.name "Rahul Dev"
git config --global user.email
Beginner
Apni email set karo. GitHub account wali email dalna best hai.
📌 Kab use karein: Username set karne ke turant baad
$git config --global user.email "rahul@example.com"
git config --list
Beginner
Saari Git settings dekho — name, email, editor sab.
📌 Kab use karein: Jab check karna ho ki settings sahi hain ya nahi
$git config --list
git config --global core.editor
Beginner
Default editor set karo commit messages ke liye.
📌 Kab use karein: VS Code ya koi aur editor use karna ho
$git config --global core.editor "code --wait"
📁
Repository Init & Clone
Nayi repo shuru karo ya kisi ki copy lo — starting point
git init
Beginner
Nayi Git repository banao current folder mein. Ek hidden .git folder create hota hai.
📌 Kab use karein: Jab bilkul naya project shuru karo aur Git tracking chahiye
$mkdir my-project && cd my-project
$git init
git clone
Beginner
Kisi existing repo ki copy apne machine pe lo. GitHub/GitLab ka URL use hota hai.
📌 Kab use karein: Team ke kisi project mein join karo ya open source contribute karo
$git clone https://github.com/user/repo.git
# Custom folder name ke saath
$git clone https://github.com/user/repo.git my-folder

Staging & Commits
Roz ka kaam — changes track karo aur save karo
git status
Daily Use
Current state dekho — kaunse files changed hain, kaunse staged hain, kaunse untracked hain.
📌 Kab use karein: Kuch bhi karne se pehle — situation samajhne ke liye
$git status
# Short format mein dekhna ho toh
$git status -s
git add
Daily Use
Files ko staging area mein daalo — matlab commit ke liye ready karo. Soch lo ye "cart mein daalna" hai.
📌 Kab use karein: Commit karne se pehle specific ya saari files select karo
$git add index.html       # specific file
$git add src/           # poora folder
$git add .              # saari files
git commit
Daily Use
Staged changes ko permanently save karo history mein. Message daalna zaroori hai.
📌 Kab use karein: Ek feature/fix complete ho jaye toh commit karo
$git commit -m "Add login page"
# Add + commit ek saath (tracked files)
$git commit -am "Fix typo in header"
git commit --amend
Important
Last commit badlo — message ya files dono. Push karne se pehle hi use karo!
📌 Kab use karein: Last commit mein typo ho ya koi file bhool gaye ho
$git commit --amend -m "Sahi message"
# File add karke amend karo
$git add forgotten.js
$git commit --amend --no-edit
🌿
Branching
Alag alag features pe parallel kaam karo bina main code touch kiye
git branch
Daily Use
Saari branches dekho ya nayi banao. Current branch ke aage * hota hai.
📌 Kab use karein: Branches list karna ho ya nayi feature branch chahiye ho
$git branch            # list branches
$git branch feature-login # nayi branch
$git branch -d old-branch  # delete
$git branch -a           # remote bhi
git checkout / git switch
Daily Use
Branch badlo. Modern Git mein switch prefer karo, checkout purana hai.
📌 Kab use karein: Ek feature se doosre feature pe jaana ho
$git switch feature-login
# Nayi branch banao + switch ek saath
$git switch -c feature-payment
# Purana tarika
$git checkout -b feature-payment
git branch -m
Beginner
Branch ka naam badlo. Current branch ka naam galat ho gaya toh.
📌 Kab use karein: Branch naam mein typo ho ya convention follow karna ho
# Current branch rename karo
$git branch -m new-name
# Kisi bhi branch ko rename
$git branch -m old-name new-name
🔀
Merging & Rebasing
Alag branches ka kaam combine karo — team ka code integrate karo
git merge
Important
Kisi branch ka kaam current branch mein lao. Merge commit banta hai history mein.
📌 Kab use karein: Feature complete ho jaye toh main/develop mein merge karo
# Pehle main branch pe aao
$git switch main
# Ab feature branch merge karo
$git merge feature-login
git rebase
Advanced
Branch ke commits ko nayi base pe rakh do — clean linear history milti hai merge se alag.
📌 Kab use karein: History clean rakhni ho aur main pe latest changes leni ho
# feature branch pe hote hue
$git rebase main
# Interactive rebase (commits edit karo)
$git rebase -i HEAD~3
git cherry-pick
Advanced
Sirf ek specific commit kisi bhi branch se current branch mein lao.
📌 Kab use karein: Poori branch nahi, bas ek fix/feature ka commit chahiye
# Commit hash pata karo pehle
$git log --oneline
$git cherry-pick a1b2c3d
🌐
Remote Operations
GitHub/GitLab ke saath kaam karo — team ke saath sync karo
git push
Daily Use
Local commits ko remote server pe bhejo (GitHub/GitLab). Backup bhi hota hai.
📌 Kab use karein: Kaam complete ho aur team ke saath share karna ho
$git push origin main
# Nayi branch push karo
$git push -u origin feature-login
# Force push (careful!)
$git push --force-with-lease
git pull
Daily Use
Remote se latest changes lo aur local se merge karo. fetch + merge dono ek saath.
📌 Kab use karein: Kaam shuru karne se pehle ya team ke changes lene ho
$git pull
$git pull origin main
# Rebase ke saath pull (cleaner)
$git pull --rebase
git fetch
Important
Remote ki information lo but merge mat karo. Safe tarika hai pehle dekho phir decide karo.
📌 Kab use karein: Dekhna ho ki remote pe kya naya aaya hai bina merge kiye
$git fetch origin
# Fetch ke baad diff dekho
$git diff main origin/main
git remote
Beginner
Remote connections manage karo — add, remove, list karo.
📌 Kab use karein: GitHub URL add karna ho ya change karna ho
$git remote -v                       # list
$git remote add origin https://github.com/u/r.git
$git remote remove origin          # remove
📜
Logs & History
Past dekho — kya kab hua, kaun ne kya badla
git log
Daily Use
Commit history dekho — kaun ne kab kya commit kiya. Bahut options hain formatting ke liye.
📌 Kab use karein: History review karni ho ya commit hash dhundhna ho
$git log
# Compact, beautiful format
$git log --oneline --graph --all
# Ek author ki commits
$git log --author="Rahul"
git diff
Daily Use
Changes dekho — unstaged, staged ya commits ke beech ka difference.
📌 Kab use karein: Commit se pehle exactly kya badla ye dekhna ho
$git diff                # unstaged changes
$git diff --staged        # staged changes
# Do branches ka diff
$git diff main feature-x
git show
Important
Kisi specific commit ki details dekho — kya changes the, message, author sab.
📌 Kab use karein: Specific commit mein kya tha ye samajhna ho
# Latest commit dekho
$git show
# Specific commit
$git show a1b2c3d
git blame
Advanced
File ki har line dekho — kisne kab likhi. Bug dhundhte waqt ye bahut kaam aata hai 😅
📌 Kab use karein: Bug kab aur kisne introduce kiya ye pata karna ho
$git blame index.html
# Specific lines
$git blame -L 10,20 app.js

Undoing Changes
Galti ho gayi? Ghabrao mat — Git mein sab undo hota hai
git reset
Important
Commits wapas lo ya staging clear karo. 3 modes: soft, mixed (default), hard. Careful with hard!
📌 Kab use karein: Local commits undo karne ho (push se pehle)
# Commit undo, files rakho staged
$git reset --soft HEAD~1
# Commit undo, files rakho unstaged
$git reset HEAD~1
# Sab delete ⚠️ careful!
$git reset --hard HEAD~1
git revert
Important
Commit ko safely undo karo — nayi "undo" commit banata hai. Pushed commits ke liye safe tarika.
📌 Kab use karein: Already pushed commit undo karna ho (reset mat use karo!)
# Last commit revert karo
$git revert HEAD
# Specific commit revert
$git revert a1b2c3d
git restore
Important
File ke changes discard karo ya staging se hatao. Modern replacement of checkout for files.
📌 Kab use karein: Kisi file ka kaam kharaab ho gaya, last commit pe wapas jaana ho
# Unstaged changes discard
$git restore index.html
# Staging se hatao (unstage)
$git restore --staged index.html
git clean
Advanced
Untracked files delete karo working directory se. Ye files staged bhi nahi hoti.
📌 Kab use karein: Build files, temp files ya test files cleanup karni ho
# Pehle dry run dekho kya delete hoga
$git clean -n
# Actually delete karo
$git clean -fd
🗃️
Stashing
Kaam beech mein chhodo bina commit kiye — emergency switch
git stash
Important
Current incomplete kaam ek side rakh do temporary — clean working directory milegi.
📌 Kab use karein: Beech mein koi urgent bug fix karna ho ya branch switch karni ho
# Kaam chhupao
$git stash
# Label ke saath
$git stash push -m "Login work in progress"
git stash pop / apply
Important
Chhupa hua kaam wapas lao. Pop removes stash, apply keeps it.
📌 Kab use karein: Emergency kaam ho gaya, ab apna kaam resume karo
# Latest stash wapas lao (delete bhi)
$git stash pop
# Stash list dekho
$git stash list
# Specific stash apply karo
$git stash apply stash@{2}
🏷️
Tags
Release versions mark karo — v1.0, v2.1 jaisi milestones
git tag
Important
Specific commits ko version labels lagao. Releases ke liye zaroori hai.
📌 Kab use karein: New version release karte waqt — v1.0.0, v2.0-beta
# Saare tags dekho
$git tag
# Annotated tag banao
$git tag -a v1.0.0 -m "First release"
# Tags push karo
$git push origin --tags
git tag -d
Advanced
Tag delete karo — local ya remote dono se.
📌 Kab use karein: Galat tag lag gaya ho ya version number change karna ho
# Local tag delete
$git tag -d v1.0.0
# Remote tag delete
$git push origin --delete v1.0.0

⚡ Ek Developer Ka Typical Daily Workflow
# Subah kaam shuru karo
$git pull                                 # 1. Latest code lo
